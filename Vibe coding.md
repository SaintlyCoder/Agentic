We have observed a significant transition from traditional procedural code scripts to intent-driven scripts. Intent-driven scripts differ from their predecessors by eliminating the need for meticulously written procedural code, complete with numerous 'if' statements, 'while' loops, and other constructs, let alone concerns about cross-platform implementation. Now, we can leverage models as an insulation medium. By providing a model with a prompt, it can reliably and repeatedly achieve the desired outcome through its internal tools, without requiring explicit instructions for every conceivable use case or varying language on the operating system. A notable example of this shift is "claw code," which, upon its release, was not a bash script but rather a prompt given to a model for insulation. This approach facilitates the creation of intent-driven scripts, where we prompt for a task effectively, and the model can tolerate minor inaccuracies in the code or exhibit greater flexibility in certain situations, while still enabling repeatable work in a largely deterministic manner.
